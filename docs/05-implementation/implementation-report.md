## 2. Структура серверного проекта (PCMEF)

```
ru.ncfu.autoshow
├── AutoshowApplication.java     # точка входа Spring Boot
├── control/        (C)  11 контроллеров (Auth, Vehicle, Order, Payment, TestDrive, ...)
├── mediator/       (M)  интерфейсы сервисов
│   └── impl/             реализации (@Service, @Transactional)
├── entity/         (E)  сущности с бизнес-методами
│   └── enums/            перечисления статусов/типов
├── foundation/     (F)  репозитории Spring Data JPA
├── dto/                 DTO запросов/ответов (по доменам)
├── mapper/              Data Mapper (Entity ↔ DTO)
├── security/            JWT, фильтр, RBAC, обработчики 401/403
├── exception/           бизнес-исключения + глобальный обработчик
└── config/              OpenAPI, инициализатор демо-данных
```

## 4. Реализованная функциональность

- **Роли и доступ:** CLIENT / MANAGER / ADMIN; JWT + `@PreAuthorize`.
- **Каталог:** поиск, фильтрация (марка, тип кузова, цена), пагинация, скрытие проданных,
  офлайн-кэш.
- **Карточка авто:** характеристики, фотогалерея, отзывы и средний рейтинг.
- **Покупка:** полная оплата и рассрочка (калькулятор аннуитета), резервирование авто.
- **Оплата:** имитация платёжного шлюза (создание → подтверждение), идемпотентность,
  транзакционность, обработка отказа.
- **Тест-драйв:** запись с валидацией и проверкой слота; обработка персоналом.
- **Заявки/заказы:** статусная модель, фильтр по статусу, уведомления сторонам.
- **Избранное, отзывы, уведомления** (с бейджем непрочитанных).
- **Аналитика:** панель показателей (наличие, продажи, выручка, заявки).
- **Администрирование:** CRUD каталога, управление пользователями, список проданных.
- **Профиль и настройки:** редактирование, смена темы, поддержка, соглашение.

## 5. Соответствие принципам

- **Слоистость PCMEF:** строгие зависимости сверху вниз, общение через интерфейсы.
- **Не анемичные сущности:** бизнес-методы в `Vehicle`, `Order`, `Payment`,
  `InstallmentPlan`, `User`.
- **SOLID:** единственная ответственность слоёв; зависимость от абстракций (интерфейсы
  сервисов/репозиториев); расширяемость без модификации (новые статусы/типы — через enum
  и методы).
- **Валидация:** Bean Validation на DTO (сервер) + моментальная валидация на клиенте.
- **Обработка ошибок:** глобальный `@RestControllerAdvice` → единый формат `ApiError`,
  корректные HTTP-статусы (400/401/403/404/409/422/500).
