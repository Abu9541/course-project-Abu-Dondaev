# Руководство администратора

Руководство по развёртыванию, настройке и администрированию ИС «Автосалон».

## 1. Системные требования

| Компонент | Требование |
|-----------|-----------|
| Сервер | Docker + Docker Compose; 2 ГБ ОЗУ; порт 8081 свободен |
| СУБД | PostgreSQL 16 (запускается в контейнере) |
| Клиент | Android 8.0+ (API 26+); для сборки — Android Studio, JDK 17 |
| Сеть | Клиент и сервер в одной сети (эмулятор: `10.0.2.2`; устройство: IP ПК) |

## 2. Развёртывание серверной части

```bash
cd application
docker compose up --build
```

- Поднимаются два контейнера: **PostgreSQL** и **backend** (Spring Boot).
- Backend доступен на **порту 8081**; Swagger UI: `http://localhost:8081/swagger-ui.html`.
- При первом запуске БД автоматически наполняется демо-данными (`DataInitializer`).

**Остановка / сброс:**
```bash
docker compose down        # остановить (данные сохраняются в томе)
docker compose down -v     # остановить и СБРОСИТЬ БД (демо-данные создадутся заново)
```

## 3. Запуск клиента

- Открыть `application/android` в Android Studio, запустить на эмуляторе (API 26+).
- Базовый URL задаётся в `app/build.gradle.kts` (`BASE_URL`):
  - эмулятор → `http://10.0.2.2:8081/`;
  - реальное устройство → `http://<IP-компьютера>:8081/` (та же Wi-Fi сеть, открыт порт 8081).
- Сборка APK: **Build → Build Bundle(s)/APK(s)**.

## 5. Функции администратора

- **Каталог (CRUD):** в каталоге — кнопка добавления авто; в карточке — редактирование
  (характеристики, цена, статус, ссылка на изображение) и удаление.
- **Пользователи:** вкладка «Люди» — изменение роли и блокировка/разблокировка учётных записей.
- **Проданные авто:** иконка в шапке каталога — отдельный список проданных автомобилей.
- **Аналитика:** вкладка «Аналитика» — наличие, продажи, выручка, число заявок.

## 6. Функции менеджера

- **Заявки:** вкладки «Тест-драйвы» и «Заказы» с фильтром по статусу.
  - Тест-драйв: подтвердить / отклонить / завершить.
  - Заказ: подтвердить → отметить оплату → завершить (или отменить).
- **Уведомления:** новые заявки и оплаты от клиентов.
- **Аналитика:** те же показатели, что у администратора.

## 7. Резервное копирование

Данные хранятся в томе Docker `pgdata`. Резервная копия:
```bash
docker exec autoshow-db pg_dump -U autoshow autoshow > backup.sql
# восстановление:
cat backup.sql | docker exec -i autoshow-db psql -U autoshow -d autoshow
```

## 8. Диагностика

| Симптом | Причина / решение |
|---------|-------------------|
| `failed to connect to the docker API` | Не запущен Docker Desktop — запустить и дождаться готовности |
| Клиент: «нет соединения с сервером» | Неверный `BASE_URL` или закрыт порт 8081 (брандмауэр); проверить сеть |
| Порт 8081 занят | Изменить маппинг порта в `docker-compose.yml` |
| Фото авто не загружаются | URL должен вести на сам файл изображения (или `file:///android_asset/...`) |
| Изменения демо-данных не видны | `DataInitializer` наполняет только пустую БД — выполнить `docker compose down -v` |

## 9. Безопасность эксплуатации

- Сменить `JWT_SECRET` и пароли учётных записей по умолчанию.
- В продакшене использовать HTTPS (в учебной среде — эмуляция, незащищённый HTTP к локальной сети).
- Регулярно делать резервные копии БД.
