# Оценка трудозатрат (COCOMO)

## 1. Исходные данные

Фактический объём исходного кода (по `git ls-files` + `wc -l`):

| Язык | Строк |
|------|-------|
| Java (сервер, main) | 4 911 |
| Kotlin (клиент) | 5 161 |
| SQL (схема + сиды) | 359 |
| **Итого исходного кода** | **≈ 10 072** (физических) |

Для оценки используется логический объём (без комментариев/пустых строк) ≈ **7,5 KLOC**;
для чувствительности приводится также расчёт по физическим **10 KLOC**.

## 2. Модель и режим

Применяется **базовая модель COCOMO** в режиме **Organic** (небольшой проект, один
квалифицированный разработчик, хорошо знакомые технологии):

- Трудоёмкость: **E = a · (KLOC)^b**, где a = 2.4, b = 1.05 (чел.-мес.)
- Длительность: **T = c · (E)^d**, где c = 2.5, d = 0.38 (мес.)
- Численность: **N = E / T** (чел.)

## 3. Расчёт

| Параметр | KLOC = 7.5 (логич.) | KLOC = 10 (физич.) |
|----------|---------------------:|-------------------:|
| Трудоёмкость E, чел.-мес. | 2.4 · 7.5^1.05 ≈ **19.9** | 2.4 · 10^1.05 ≈ **26.9** |
| Длительность T, мес. | 2.5 · 19.9^0.38 ≈ **7.8** | 2.5 · 26.9^0.38 ≈ **8.7** |
| Численность N, чел. | 19.9 / 7.8 ≈ **2.5** | 26.9 / 8.7 ≈ **3.1** |

## 4. Интерпретация и сверка с фактом

Базовая COCOMO моделирует **полный промышленный жизненный цикл** (анализ → проектирование
→ кодирование → тестирование → интеграция) при **средней отраслевой
производительности и командной разработке**. Поэтому она даёт «коммерческую» оценку
≈ 20 чел.-мес. и команду из 2–3 человек.

Фактические трудозатраты учебного проекта существенно ниже (≈ **125 ч ≈ 0,8 чел.-мес.**,
один разработчик за семестр — см. [WBS](wbs.md)). Расхождение объяснимо и закономерно:

- современные фреймворки (**Spring Boot, Spring Data, Jetpack Compose**) радикально
  сокращают объём «ручного» кода и труд по сравнению с допущениями COCOMO;
- значительная доля кода — декларативная (аннотации, Compose-разметка), а не алгоритмически
  сложная;
- отсутствуют накладные расходы крупной команды (координация, длинные циклы интеграции).

**Вывод.** COCOMO подтверждает, что по объёму система соответствует полноценному
проекту (≈ 7,5–10 KLOC), а достигнутая за семестр трудоёмкость — результат высокой
продуктивности выбранного технологического стека. Расчёт носит демонстрационный характер
(владение методикой оценки), а не директивный.

## 5. Оценка стоимости (для ROI)

При коммерческой ставке 160 000 ₽/чел.-мес. и реалистичной для учебного проекта
трудоёмкости ≈ 3 чел.-мес. стоимость разработки ≈ **480 000 ₽** — это значение
использовано в [экономическом обосновании (ROI)](../00-initiation/economic-justification.md).
