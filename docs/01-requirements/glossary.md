# Глоссарий (расширенный, системный)

Расширяет [бизнес-глоссарий](../00-initiation/business-glossary.md) техническими терминами
уровня системы и реализации.

| № | Термин | Определение |
|---|--------|-------------|
| 1 | **PCMEF** | Архитектурный паттерн из 5 слоёв: Presentation, Control, Mediator, Entity, Foundation. Зависимости направлены строго вниз. |
| 2 | **Presentation (P)** | Слой представления: на клиенте — Compose-экраны и ViewModel; отвечает за UI и ввод. |
| 3 | **Control (C)** | Слой управления: REST-контроллеры; принимает запросы, валидирует, делегирует в Mediator. |
| 4 | **Mediator (M)** | Слой бизнес-логики: сервисы, транзакции, бизнес-правила. |
| 5 | **Entity (E)** | Слой сущностей: бизнес-объекты с состоянием и бизнес-методами (не анемичные). |
| 6 | **Foundation (F)** | Слой доступа к данным: репозитории (Spring Data JPA). |
| 7 | **REST API** | Стиль межсетевого взаимодействия клиент-сервер по HTTP с ресурсами и стандартными методами. |
| 8 | **Эндпоинт** | Конкретный URL+метод REST API (например, `POST /api/orders`). |
| 9 | **JWT** | JSON Web Token — подписанный токен для stateless-аутентификации. |
| 10 | **RBAC** | Разграничение доступа на основе ролей (CLIENT/MANAGER/ADMIN). |
| 11 | **BCrypt** | Алгоритм адаптивного хеширования паролей. |
| 12 | **DTO** | Data Transfer Object — объект передачи данных между слоями/по сети (отделён от сущности). |
| 13 | **Data Mapper** | Паттерн, отделяющий доменные объекты от способа их хранения (мапперы + репозитории). |
| 14 | **Identity Map** | Паттерн уникальности объекта в пределах сессии (реализован контекстом персистентности Hibernate). |
| 15 | **ORM** | Object-Relational Mapping — отображение объектов на реляционные таблицы (Hibernate/JPA). |
| 16 | **Repository (репозиторий)** | Интерфейс доступа к коллекции сущностей (Spring Data JPA). |
| 17 | **OpenAPI / Swagger** | Спецификация и UI для документирования и проверки REST API. |
| 18 | **StateFlow** | Поток состояния (Kotlin Coroutines), на который подписывается UI. |
| 19 | **ViewModel** | Компонент Android, хранящий состояние экрана и переживающий пересоздание UI. |
| 20 | **Retrofit** | HTTP-клиент для Android, генерирующий реализацию API по интерфейсу. |
| 21 | **Room** | Библиотека локального хранилища (SQLite) для офлайн-кэша. |
| 22 | **DataStore** | Хранилище пар «ключ-значение» для настроек и токена сессии. |
| 23 | **Идемпотентность** | Свойство операции давать одинаковый результат при повторном выполнении (важно для оплаты). |
| 24 | **Платёжный шлюз (имитация)** | Серверный компонент, воспроизводящий протокол эквайринга: создание платежа → подтверждение. |
| 25 | **Транзакция** | Атомарная единица работы с БД (`@Transactional`): либо все изменения применяются, либо ни одного. |
| 26 | **Миграция статусов** | Конечный автомат жизненного цикла заказа/тест-драйва (например, `PENDING → CONFIRMED → PAID → COMPLETED`). |
| 27 | **3НФ** | Третья нормальная форма — критерий нормализации схемы БД. |
| 28 | **JaCoCo** | Инструмент измерения покрытия кода тестами. |
